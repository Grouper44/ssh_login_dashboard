# 第三階段：排程 / 預測 / 模擬分析

## 概述

本階段基於第一、二階段收集的歷史資料，進行排程優化、趨勢預測與情境模擬，完成 MES 完整流程：

**資料收集 → 即時監控 → 排程優化 → 預測分析 → 情境模擬**

## 檔案

- `phase3_analysis.ipynb` — Jupyter Notebook，包含所有分析與互動式模擬

## 功能說明

### 1. 排程建議 — 最佳使用時段

- 將每日登入量與 24 小時分布交叉，產生 **7×24 負載矩陣**
- 以熱力圖呈現（綠色 = 低負載，紅色 = 高負載）
- 自動找出並推薦 **負載最低的 5 個時段**

**使用資料：** `hourStats`（24小時登入分布）+ `weekStats`（7天每日登入次數）

### 2. 趨勢預測 — 未來 3 天登入量

- 使用 **加權移動平均 (Weighted Moving Average)**
  - 近 3 天佔 70% 權重，遠期佔 30%
  - 預測值遞迴作為下一天輸入
- 圖表包含歷史實線、預測虛線、信賴區間
- 顯示趨勢方向（↑ 上升 / ↓ 下降 / → 持平）

**使用資料：** `weekStats`（過去 7 天登入次數）

### 3. What-if 資源模擬

互動式模擬面板，可調整以下參數：

| 參數 | 範圍 | 說明 |
|------|------|------|
| 同時在線人數 | 1 ~ 20 | 模擬不同使用者數量 |
| 活躍容器數 | 1 ~ 8 | 模擬增減容器 |
| GPU 任務強度 | 低 / 中 / 高 | 影響 GPU 佔用比例 |

根據活躍容器的歷史平均 CPU/MEM/GPU 使用率，即時計算並顯示：
- 三個環形圖（CPU / MEM / GPU 預估使用率）
- 安全評估：≤60% 安全 / 60-85% 警告 / >85% 危險

**使用資料：** `containers`（容器即時資源數據）

## 資料來源

Notebook 直接從 Google Sheets 抓取真實資料（透過公開 CSV 匯出）：
- **工作表1**：SSH 登入紀錄（時間 / 使用者 / IP位置）
- **容器進入紀錄**：容器進入紀錄（時間 / 使用者 / 容器ID）

不使用假資料，所有統計、預測、模擬的基準值皆來自真實歷史紀錄。

## 執行方式

```bash
pip install numpy matplotlib ipywidgets pandas
jupyter notebook phase3_analysis.ipynb
```

## 功能對照表

| 功能 | 方法 | 輸入資料 | 輸出 |
|------|------|----------|------|
